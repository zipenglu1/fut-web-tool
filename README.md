新增了 高级混合联赛SBC 挑战自动循环功能，可在挑战列表页一键启动自动处理流程。
<img width="742" height="632" alt="image" src="https://github.com/user-attachments/assets/4300930c-914f-42d2-b7a6-46812d9f2702" />

点击“循环完成”则开始自动循环完成SBC
自动流程为：

1. 自动寻找当前未完成挑战
2. 进入挑战
3. 执行 `SBC方案填充`
4. 执行 `一键替换所有假想球员`
5. 满足条件时自动提交
6. 返回并继续处理下一个挑战

## 自动跳过无法完成的挑战

为了避免脚本卡死在某个挑战上，新增了跳过机制：

- 如果挑战仍有假想球员未替换成功，则临时跳过
- 如果当前阵容无法提交，则临时跳过
- 如果存在不可提交球员，也会跳过当前挑战

这样可以保证脚本优先继续处理其他可完成的挑战，而不是反复卡在同一项。

## 自动检索未完成挑战

循环过程中不再死盯已经完成的挑战，而是会实时检索当前未完成的挑战继续处理。

这项改动避免了：

- 重复进入已完成挑战
- 因缓存未刷新导致的错误循环
- 已完成项目被反复尝试的问题

## 循环结束条件优化

本次还调整了自动循环的结束策略：

- 如果一整轮挑战都已完成，则可以继续进入下一轮
- 如果一整轮结束后仍然存在未完成挑战，则自动停止循环

这样可以避免脚本无限卡在无法完成的挑战上。

## 优化一键填充的逻辑处理方式

现在的一键替换整体可以概括为：

1. 先循环替换假想球员。
2. 如果当前已替换上的真人球员提供的有效默契还不够，就尝试进行一次位置重排。
3. 一旦默契已达标，后续替换优先走省资源。
4. 当场上已经没有假想球员后，再做一次最终降分复检。
5. 如果降分后仍满足全部 SBC 条件，就保存更省的阵容。
